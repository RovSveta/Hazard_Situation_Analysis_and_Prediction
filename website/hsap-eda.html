01 Dataset Overview
Analysis

SCOPE OF THE PROJECT
The project started with a task to analyze data from a company called "Trionas". The data is from a truck with a dedicated button that the driver is instructed to press when a hazardous event happens.
The data used is a collection of hazard events collected over a specific period. Approximately 890 events were collected over a 3-month period.
The instructions given on what to do with said data were to perform exploratory data analysis and generate insights about the patterns and trends in the hazard events.
This was possible due to having also sensor data from the truck including 3 movement sensors and its GPS-location. Additionally we also had road surface data to make insights easier.

ANALYSIS

EMERGING ISSUES DURING THE PROJECT
During the project a few issues came up that made the analysis more difficult. The main issue was that the data had so many rows of a button being pressed with some gaps being below 2 seconds.
This made it difficult to analyze the data as it was not clear if these were multiple events or if they were just one event with multiple signals.
We believe this was caused by latency issues in the BLE broadcast interval configurations.

Also a problem, where we cannot know what happened a while before or after the button was pressed, makes it harder to go deeper into every event. This way having more data about the bofres and afters of an event would make clearer analysis easier on every event.
