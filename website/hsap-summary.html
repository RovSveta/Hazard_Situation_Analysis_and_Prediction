01 Project Scope
Overview

This project began with a real-world challenge provided by Trionas: to analyze the data collected from a truck equipped with a special button. Drivers were instructed to press this button whenever they experienced a potentially dangerous situation on the road.

The dataset came from one truck operating in Southwest Finland, mainly on routes between Turku, Salo, and Helsinki. Data was collected during three driving days: November 10, November 27, and January 9.

Over a three-month period, around 890 hazard events were recorded. The main objective was to explore this data, identify meaningful patterns, and uncover trends that could help improve road safety and driving conditions.

To support the analysis, the dataset also included valuable additional information such as:

  • Three motion sensors – measuring acceleration, temperature, and humidity
  • MD30 road surface sensor – measuring grip, ice, water, snow, and surface temperature
  • GPS tracking – providing exact event locations

By combining these data sources, we were able to better understand when, where, and why hazardous events happen.

02 Hazard Button Analysis Results
Key Findings
  • Original dataset contained 890 recorded button signals
  • Many records were duplicate Bluetooth transmissions after a single press
  • Records were grouped into separate event sessions using a 5-second time gap rule
  • If more than 5 seconds passed between two signals, it was counted as a new button press event
  • Final cleaned dataset identified 25 real hazard button presses
  • These events represent actual situations where the driver reported danger or discomfort
Outcome

The cleaned hazard button data provided a reliable event timeline for the project. These confirmed hazard events were later combined with motion sensors, road condition data, GPS locations, and machine learning models to better understand risky driving situations.

Motion Sensor Analysis
  • Three motion sensors were installed in different truck locations
  • Sensor orientation and signal strength helped estimate mounting positions inside the vehicle
  • One sensor was likely inside the cabin, while others were mounted closer to the truck body
  • Strong vibration patterns were detected in one sensor, indicating areas with higher vehicle movement
  • Bluetooth signal variation was identified as normal due to the metal truck cabin environment
  • Incorrect temperature readings were detected and corrected during data cleaning
03 Road Surface Analysis
Key Findings
  • November 10: Dry roads with high grip and safe conditions
  • November 27: Conditions changed from dry to wet, with grip dropping significantly during the trip
  • January 9: Icy and slippery roads with very low grip levels
Data Merging & Event Matching
  • The original hazard button data was cleaned and combined with motion sensor and road surface data
  • Sensor values were matched using a 60-second time window around each button press
  • Some events had identical sensor readings because the Bluetooth button continued sending signals for up to 60 seconds after one press
  • These duplicate events were merged into single hazard events for better accuracy
  • Missing values were completed using the closest available sensor data
Outcome

After final processing, the dataset contained 18 confirmed hazard events linked with vehicle movement and road conditions, ready for visualization and machine learning analysis.

04 Interactive Hazard Map
Key Features
  • Built using Folium to create an interactive web-based map
  • Visualized all confirmed hazard events using GPS location data
  • Combined location data with speed, road grip, and day/night conditions
  • Included clickable markers with detailed event information
  • Added date and time filters for easier data exploration
Outcome

The map provides a clear visual overview of where hazard events happened and under what conditions.

05 Hazard Analysis Chatbot (Small Language Model)
Highlights
  • Real hazard event data was used as input, including speed, acceleration, friction, and road conditions
  • Powered by SmolLM2-1.7B to analyze each event
  • Generated clear explanations of possible dangerous situations
  • Considered factors such as speed, movement, weather, and darkness
  • Built with Gradio as an interactive chatbot interface
Outcome

The chatbot transforms sensor data into easy-to-understand safety insights for users.
